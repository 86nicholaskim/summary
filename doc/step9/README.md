# 🧩 Step 9: 데이터 영속성 및 고도화 (Persistence & RAG)

사만다가 과거의 데이터를 기억하고 **더 똑똑하게 문서를 관리**하는 최종 단계입니다.

---

## 🎯 목표
1. **DB 통합**: 로컬 파일을 넘어 PostgreSQL을 이용한 전문 문서 관리.
2. **지식 베이스(RAG)**: 사용자의 과거 템플릿과 스타일을 학습하여 제안.
3. **자율 워크플로우 완성**: "어제 찍은 그 서식대로 오늘 것도 만들어줘" 수준의 자동화.

## 📝 세부 작업 리스트
- [ ] **9.1 DB Persistence**: Prisma ORM 및 PostgreSQL 연동.
- [ ] **9.2 Personalization**: 사용자의 선호 스타일 및 템플릿 학습.
- [ ] **9.3 RAG Engine**: 과거 문서 데이터를 기반으로 한 지능형 검색 및 제안.
- [ ] **9.4 Final Audit**: 전체 시스템 안정성 검수 및 최적화.

## 💡 핵심 기술
- PostgreSQL & Prisma
- Vector Database (선택 사항)
- RAG (Retrieval-Augmented Generation)
