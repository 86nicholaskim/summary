🌀 웹사이트 관리 전체 워크플로우 (1~2단계 요약)

1. 개발 환경 기동 (pnpm & Turbo)개발자는 여러 명령어를 칠 필요 없이 하나의 명령어로 로컬 서버를 켭니다.
   - 실행: pnpm dev
   - 동작: Turbo가 type-check(tsc)와 build:watch(rollup)를 동시에 실행합니다.
   - 효과: 코드 수정 시 타입 에러를 즉시 확인하면서, 동시에 빌드된 결과물이 dist/에 실시간으로 업데이트됩니다.

2. 코드 작성 및 레이어 분리 (src/)
   가장 많이 사용하는 레이어드 아키텍처 패턴에 따라 코드를 배치합니다.

- Route: URL 경로 설정 (/users, /posts 등)
- Controller: 들어온 요청을 서비스로 전달하고 응답을 보냄.
- Service: 실제 비즈니스 로직 및 DB 처리 (가장 중요한 부분).
- 효과: 로직이 분리되어 있어 특정 기능 수정 시 해당 파일만 고치면 되므로 유지보수 속도가 비약적으로 상승합니다.

3. 코드 품질 자동화 (Lint & Format)코드 스타일로 인한 논쟁을 원천 차단합니다.

- Format: VSCode에서 저장(Ctrl+S) 시 Prettier가 공백, 따옴표 등을 자동 교정.
- Lint: ESLint가 잠재적인 문법 에러나 권장되지 않는 패턴을 실시간으로 경고.
- 효과: 누가 코드를 짜더라도 같은 스타일을 유지하며, 사소한 실수를 배포 전에 잡아냅니다.

4. 빌드 파이프라인 (Rollup & Turbo)운영 환경에 올리기 위한 최종 결과물을 만듭니다.

- 실행: pnpm build
- 동작: Turbo가 캐시를 확인한 후, 변경된 부분만 Rollup을 통해 dist/ 폴더에 단일 ESM 파일들로 번들링합니다.
- 효과: 불필요한 파일을 제거하고 최적화된 결과물만 남겨 서버 실행 속도와 배포 효율을 높입니다.
